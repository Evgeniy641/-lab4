Обзор проекта
Был разработан консольный проект на C#, содержащий статический класс LabTasks с реализацией методов для работы с различными типами коллекций. Проект охватывает следующие структуры данных:

List - операции с динамическими массивами

LinkedList - работа со связными списками

HashSet - операции с множествами

Dictionary - работа со словарями и обработка файлов

Реализация
Класс LabTasks
Содержит методы для каждой задачи, организованные по типам коллекций. Все методы включают проверку входных данных и обработку исключительных ситуаций.

Метод Main
Реализует автоматическое тестирование всех заданий с демонстрацией результатов работы каждого метода.

Функциональность
Задание 1.2: List - Замена первого вхождения
Метод: ReplaceFirstOccurrence<T>(List<T> L, List<T> L1, List<T> L2)

Назначение: Заменяет первое вхождение последовательности L1 в списке L на последовательность L2

Тесты:

Входные данные: L = [1, 2, 3, 4, 5, 2, 3, 6], L1 = [2, 3], L2 = [9, 9, 9]

Результат: [1, 9, 9, 9, 4, 5, 2, 3, 6]

Обработка граничных случаев: пустые списки, отсутствие вхождения

Задание 2.2: LinkedList - Сортировка
Метод: SortLinkedList<T>(LinkedList<T> list) where T : IComparable<T>

Назначение: Сортирует элементы связного списка в естественном порядке

Тесты:

Входные данные: [5, 2, 8, 1, 4]

Результат: [1, 2, 4, 5, 8]

Обработка: списки из 0-1 элемента, проверка на null

Задание 3.2: HashSet - Анализ игр студентов
Класс: StudentGameData
Метод: AnalyzeStudentGames(List<StudentGameData> students, HashSet<string> allGames)

Назначение: Анализирует какие игры популярны среди студентов

Анализируемые категории:

Игры, в которые играют все студенты

Игры, в которые играют некоторые студенты

Игры, в которые не играет никто

Тестовые данные:

Студенты: Иванов, Петров, Сидоров, Козлов, Смирнов

Игры: Dota 2, CS:GO, Valorant, Minecraft, GTA V, Fortnite и др.

Задание 4.2: HashSet с файлом - Глухие согласные
Метод: PrintVoicelessConsonantsNotInSomeWord(string filePath)

Назначение: Находит глухие согласные, которые не входят хотя бы в одно слово текста

Алгоритм:

Чтение и разбор текста из файла

Определение множества всех глухих согласных русского языка

Поиск согласных, отсутствующих хотя бы в одном слове

Сортировка и вывод результата

Задание 5.2: Dictionary - Олимпиада
Класс: Participant
Метод: FindOlympiadWinners(string filePath)

Назначение: Определяет победителей олимпиады по максимальному баллу

Формат файла:

Первая строка: количество участников

Последующие строки: Фамилия Имя Балл1 Балл2 Балл3

Функциональность:

Чтение и парсинг данных из файла

Расчет общего балла для каждого участника

Определение победителей с максимальным баллом

Детализированный вывод результатов

Особенности реализации
Обработка ошибок
Проверка входных параметров на null

Валидация формата данных в файлах

Обработка исключений FileNotFoundException, FormatException

Грациозная обработка некорректных данных

Вспомогательные методы
CreateTestStudents() - создание тестовых данных студентов

CreateTestGames() - создание тестового набора игр

CreateOlympiadTestFile() - генерация тестового файла олимпиады

CreateTextTestFile() - создание тестового текстового файла

Кодировка и локализация
Использование UTF-8 для работы с русским текстом

Правильная настройка кодировки консоли

Поддержка кириллических символов во всех операциях

Тестирование
Автоматическое тестирование
Метод TestAllTasks() выполняет комплексное тестирование всех заданий:

List - замена подпоследовательности с выводом промежуточных результатов

LinkedList - сортировка с отображением до/после

HashSet - анализ игр с группировкой по категориям

File Analysis - работа с текстовыми файлами и множествами символов

Dictionary - обработка структурированных данных из файлов

Тестовые данные
Проект включает генераторы тестовых данных для всех заданий, что обеспечивает повторяемость тестов и демонстрацию работы в различных сценариях.
